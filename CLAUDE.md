# AUREN

**Anonymous, Unique, Real, Exchange, Network**

AI 時代的真人可信交流平台。

---

## 核心信念

當 AI 生成內容變得無限便宜，真人討論會變成稀缺資產。未來網路上最有價值的東西不是更多內容，而是「我知道這句話背後有一個真實的人」。

---

## 起源

PTT 曾經是台灣網路的公共廣場。被買帳號、組織性政治滲透之後，真實台灣人的聲音被淹沒了。全球也沒有一個平台解決這個問題——Facebook 是詐騙溫床，X 是 bot 天堂，Reddit 有組織性滲透。AUREN 是為了把廣場還給真人。

---

## 產品定義

- **形式**：主題板塊式論壇（類 PTT / Reddit）
- **身份模型**：後端實名、前端匿名
  - 對平台：經過驗證的真實唯一身份
  - 對其他用戶：完全匿名
- **語言**：AI 即時翻譯作為基礎建設，語言不是維度。法國人寫法語，台灣人看中文，驗證標章跨語言成立
- **內容**：真人使用 AI 輔助寫作沒問題，責任主體還是那個真人
- **兩種內容形式**：
  - 板塊文章：發到特定板，歸屬社群討論（PTT 精神）
  - 個人 murmur：發到個人空間，可被交流（Threads 精神）

## 核心架構決策

### 驗證機制
- 手機門號作為第一階段通用基礎（全球可用、低摩擦）
- 驗證層設計為可抽換，之後可疊加更強驗證（自然人憑證、政府 ID）
- 一人一帳號是關鍵，不是身份公開

### 身份與內容分離
```
[驗證服務]              [論壇服務]
手機號碼                只認識 Token
    ↓                        ↓
  Token ←————————→  發文、留言、板塊
```
Token 有時效性，過期後連平台自己都無法把內容連回真人。

### 法律與政治保護
- 公司實體設在境外（美國 Delaware 或新加坡）
- 身份驗證資料存放境外伺服器
- 即使台灣政治情況改變，資料無法被直接取得
- 真正的犯罪透過境外民主法律框架處理
- 政治迫害因為司法獨立性而難以成立

### 驗證等級制度（仿加密貨幣交易所模型）

| 等級 | 驗證方式 | 權限 |
|------|---------|------|
| Level 1 | 手機門號 | 每日發文/回覆有上限，不能進敏感板塊 |
| Level 2 | 裝置綁定 | 發文上限提高，可進一般板塊 |
| Level 3 | 支付驗證 | 無限發文，可進敏感板塊、投票 |
| Level 4 | 政府 ID | 最高信任標章，可申請開板、擔任板主 |

- 所有等級免費，用驗證換權限而非用金錢
- 數字參數（每日上限幾篇）之後調整，現在不需要定
- 天然限制組織性灌水：大量假帳號停在 Level 1，操控成本大幅提高

### 治理架構
```
平台 ToS（底線規則）
    ↓
各板板規（板主自訂文化）
```
平台只管真正不能碰的底線，其他讓板主和社群決定。

## 品牌設計

### Logo
- 字型：AUREN，全大寫，字重 900
- **R 固定用品牌主色 #F5A623**（代表 Real，核心價值的視覺錨點）
- 其他字母跟隨 theme 顏色

### 主題系統
- 用戶可自選 theme（深色 / 淺色 / 暖色）
- 品牌色 #F5A623 不被 theme 吃掉，永遠固定
- Theme 只影響介面底色、卡片色、文字色

### 暱稱顏色（依驗證等級）
- 概念：信任資訊融入閱讀體驗，不用額外標章
- Level 4（政府 ID）使用品牌主色 #F5A623
- Level 1-3 的具體顏色待定

## 商業模式方向

靠「真實性」賺錢，不靠用戶數。商業模式與產品價值觀對齊：假帳號傷害產品，平台有真正的動機去清除。

具體形式待定，方向包括：
- 向用戶收費（付費進入真人空間）
- 向企業收費（真人調查、市場研究）
- 真人認證作為 API 基礎建設

## Dev Notes（給 Claude）

### 環境限制
- 伺服器 Node.js 版本是 **v14**，drizzle-kit 需要 v16+，`npm run db:generate` 和 `npm run db:migrate` 會因為語法錯誤失敗
- **新增 migration 的正確做法**：手動寫 SQL 檔案到 `lib/db/migrations/`，用 psql 直接執行，例如：
  ```bash
  PGPASSWORD=auren_dev psql -h localhost -U mark -d auren -f lib/db/migrations/<file>.sql
  ```
- 資料庫：`postgresql://mark:auren_dev@localhost:5432/auren`
- Redis：`redis://localhost:6379`（只用在 OTP rate limiting）

### 架構摘要
- SSE (`lib/sse.ts`)：in-process Map，單一 Node.js instance 夠用，多 instance 要換 Redis Pub/Sub
- 頻道命名：`post:{id}`（回覆）、`board:{slug}`（新文章、upvote、刪除）
- 所有 SSE route 都要加 `export const dynamic = 'force-dynamic'`
- SWR + `useSWRInfinite`：所有列表頁統一用 cursor-based 分頁，sentinel div + IntersectionObserver 自動載入

### 資料庫 Schema 重點
- `users.id` = `userTokens.id`（token 就是用戶 ID，過期後無法回溯）
- `posts.status` / `murmurs.status`：soft delete 用 `'deleted'`，查詢一律 `eq(status, 'active')`
- Upvote 表：`post_upvotes`（已建）、`murmur_upvotes`（已建，migration 0002）
